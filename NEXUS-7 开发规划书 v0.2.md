## 1. 设计哲学与产品定位

### 1.1 一句话定位

通过 AI 目标规划、虚拟小人陪伴、游戏化成长、行为数据追踪、复盘反馈和奖励惩罚机制，把现实人生包装成一个可升级、可反馈、可调整的"小说系统"。

### 1.2 产品灵魂（不可妥协）

**核心命题**：
> 系统要持续判断：宿主今天的行为，是在接近理想人生，还是远离理想人生。

所有功能围绕这一个判断展开：
- 任务系统记录行为
- 目标系统定义方向
- 复盘系统纠偏
- 积分系统提供动力
- 虚拟小人提供人格化反馈
- AI 中枢负责理解和决策
- 外星科技感增强沉浸与信任

### 1.3 五条不可妥协的原则

1. **AI 是主语，不是工具**
   AI 主动观察、提问、提议、复盘；用户在对话中校准方向。任务记录是 AI 的副产物，不是用户的负担。

2. **记忆是核心资产**
   一个不记得你的 AI 不可能帮你成长。系统必须沉淀目标演化、决策模式、情绪曲线、能量节律、长期承诺。

3. **闭环胜过功能**
   "目标 → 行动 → 反馈 → 校准"闭环必须严丝合缝。一个能让用户三个月后回头看到自己变化的系统，胜过 50 个 Beta 功能。

4. **诚实胜过讨好**
   AI 不是用来鼓励你的，是用来识别你的"伪完成"、"冲动目标"、"自欺欺人"的。**温柔的镜子不能反映现实，必须是冷静的镜子**。

5. **沉浸感是工程问题**
   外星科技感、虚拟小人、权限解锁、仪式化交互——这些不是装饰，而是经过设计的心理学装置，目的是让用户与系统建立"宿主-系统"的深度绑定关系。

### 1.4 反模式（明确不做什么）

| 反模式 | 为什么不做 |
|---|---|
| 粗糙的打卡返现式游戏化 | 长期侵蚀内在动机；本系统采用属性/衰减/可信度等深度内部生态 |
| AI 只会鼓励、不会挑战用户 | 退化为 ChatGPT 套壳；本系统必须有"目标真实性检测""自欺识别""黑箱裁决" |
| 用一个超大 Prompt 解决一切 | 不可调试、不可演进、不可控成本；必须 Agent + 工具调用 |
| 强制 SOP / 模板化任务结构 | 个人成长高度个性化；模板异化用户 |
| 数据接入只为展示，不参与决策 | 接入手表/IoT 不是炫技；必须用于自欺识别和路径优化 |
| 把虚拟小人做成皮肤 | 小人是人格化载体，性格必须由 AI 自主调整，不由用户配置 |

## 4. 11 个核心子系统详细设计

> 任务管理系统.md 列出了 10 个子系统，本规划书新增 **系统进化引擎** 作为第 11 个，让 AI 自主进化有显式承载。

### 4.1 子系统 1：宿主档案系统

**职责**：建立并演化宿主的完整画像。

**核心组件**：
- 基础信息表（身份、状态、能力、时间、性格、激励偏好、长期目标）
- 初始测评流程（首次启动仪式化的对话+问卷混合）
- 档案演化机制（Profile Agent 每周扫描事件流，自动提议更新；用户确认或拒绝）

**关键设计**：
- 档案不是一次填好就不变，而是**渐进演化**
- 用户随时可看、改、删任意条目（数据主权）
- 红线机制：用户标记的"绝不"事项（如不熬夜、不吃糖）会被 AI 持续监控

**接入测评内容**：
- 性格 5 大维度（OCEAN）
- 激励类型（成就/隶属/权力/自治）
- 24 项性格优势（VIA）
- 时间偏好（晨型/夜型/混合）

### 4.2 子系统 2：目标规划系统

**职责**：从模糊愿望生成可执行的五层目标体系。

**核心组件**：
- 五层目标结构（终极→长期→阶段→周→日）
- **模糊目标明确化**对话引擎（多轮苏格拉底式追问）
- 智能路径生成器（基于宿主状态生成个性化路径）
- **目标真实性检测**（高级功能）

**关键设计**：
- 用户说"我想变得更自律"→ AI 多轮追问 → 重构为"30 天内每日学习 2 小时 + 短视频 1 小时内"
- 每个目标生成时同步生成"冲动设定概率"评分
- 目标可被冻结、降级、重启
- 父子目标自动联动（子目标进度推送到父目标）

**Prompt 设计要点**：
- 规划 Agent 注入：宿主档案 + 当前所有目标 + 最近 30 天事件流 + 能量节律
- 输出必须包含"我建议这样拆解的原因"+ "潜在风险"+ "前置条件"

### 4.3 子系统 3：任务执行系统

**职责**：管理日常任务的整个生命周期。

**核心组件**：
- 任务对象（9 字段：名称、所属目标、难度、预计时间、能量消耗、奖励积分、失败惩罚、验收标准、证明方式）
- 任务类型（学习/工作/健康/整理/反思/社交/娱乐）
- 任务状态机（未开始→准备中→进行中→暂停→已完成→已复盘 + 失败路线）
- **任务来源多样化**：用户主动创建 / AI 提议 / 重复任务 / 数据驱动自动创建

**关键设计**：
- 娱乐也是任务，纳入系统而不是禁止
- 任务**超额完成**有额外奖励
- 任务**多次失败**触发系统自动降难度（不是失败→放弃，而是失败→重新设计）
- 每个任务保留完整状态变迁日志（用于行为分析）

### 4.4 子系统 4：虚拟小人系统

**职责**：以人格化形象承载系统对宿主的反馈与陪伴。

**核心组件**：
- 5 类小人（主系统 / 学习 / 健康 / 娱乐 / 复盘）
- 8 种状态（待机/专注陪伴/提醒/庆祝/失望/严厉/关怀/进化）
- 性格参数（亲密度、严厉度、信任度）
- **性格 AI 自主调整**（用户连续达标→变信任；长期逃避→进入冷静审判模式）
- 多端可视化（桌面悬浮、手机 Widget、锁屏通知、App 首页）

**关键设计**：
- Phase 1 先做**主系统小人**，其他 4 个 Phase 2+ 逐步加入
- 小人台词由 Companion Agent 实时生成，融合当前情境（任务进展、情绪、时间、最近事件）
- 小人对话有"叠加状态"：例如"严厉+关怀"混合，避免单调
- 小人会**记得它对宿主说过的话**（融入语义记忆）

### 4.5 子系统 5：游戏化成长系统

**职责**：将长期成长转化为可见的属性、等级、经验。

**核心组件**：
- 觉醒等级（Lv.1 未觉醒 → Lv.100 完全体）
- 6 维属性（智力/体力/专注力/意志力/创造力/秩序感）
- 经验条（每个属性独立成长）
- 衰减机制（3 天不做停止增长 / 7 天轻微衰减 / 14 天提醒 / 30 天目标冻结）
- 隐藏属性：**系统可信度**（用户逃避复盘、伪完成→可信度下降→AI 提供建议时附带置信度降级）

**关键设计**：
- 等级公式：`等级 = 目标完成度 + 属性成长 + 连续性 + 复盘质量 + 长期稳定性`，权重可调
- 属性增长由 AI 根据任务内容判断（不是用户预设），避免刷分
- 衰减"轻微"——不是惩罚，而是提醒
- 系统可信度公开可见，用户能看到"自己在多大程度上对系统诚实"

### 4.6 子系统 6：积分与商城系统（能量点系统）

**职责**：外部激励 + 现实与虚拟奖励兑换。

**核心组件**：
- 能量点（积分）来源（任务完成/连续打卡/高质量复盘/主动抵抗诱惑）
- 三类商城（现实/虚拟/功能）
- **AI 自动推荐商城奖品**（Phase 3+）
- 防刷机制（主观/数据/成果/AI 判断）

**关键设计**：
- 现实奖励需要用户**实际兑换**（系统提醒、用户自己执行）
- 虚拟奖励（小人皮肤、UI 主题）可立即生效
- 功能奖励与权限解锁系统打通
- 抵抗诱惑积分要求"诱惑被识别"：例如夜晚 22:00 打开短视频 App 又主动关闭 → AI 检测到 → 触发奖励

### 4.7 子系统 7：复盘系统

**职责**：让系统真正变聪明的关键。

**核心组件**：
- 每日复盘（主观+客观数据对比）
- 每周总结（中等规模调整）
- 每月报告（大规模调整，包装为"宿主进化报告"）
- 季度/年度复盘（重大方向调整）

**关键设计**：
- **主观-客观对比**是核心机制：用户说"今天很废"但客观数据显示完成 3 任务、专注 2.3 小时 → AI 反驳并安慰
- 复盘的提问由 Coach Agent 生成，不重复（每次复盘提问 3-5 个，从 50+ 候选问题中智能选择）
- 月报告必须包含"系统对宿主的理解更新"小节
- 复盘逃避会触发可信度下降 + 小人状态变化

### 4.8 子系统 8：决策辅助系统

**职责**：实现"判断今天做的事让宿主变好还是变差"。

**核心组件**：
- 行为影响评分（每个行为打 -10 到 +10 的影响分）
- 每日净成长值
- **选择前预测**（用户犹豫时主动咨询系统）
- **人生路线模拟**（高级功能，Phase 3+）

**关键设计**：
- 评分由 Decision Agent 实时计算，结合宿主档案的个人化权重
- 选择前预测必须给出**多方案对比**（A/B/C 三方案，含短期愉悦 / 明日影响 / 系统建议）
- 路线模拟使用 Opus 4.7 + 长上下文（注入过去 6 个月事件流）
- **黑箱裁决机制**：系统可以选择不给完整解释（"当前权限不足，无法展开"），用于增强神秘感，但重大决策必须可解释

### 4.9 子系统 9：外星科技沉浸系统

**职责**：营造"小说系统感"的全套心理学装置。

**核心组件**：
- 系统世界观（NEXUS-7 = 来自未知文明的个体进化协议）
- UI 风格关键词（深空/量子/扫描/全息/权限/宿主/进化/校准/协议/数据流）
- 文案改写表（普通→外星科技版）
- 首次启动仪式（10 步绑定流程）
- 权限解锁系统（Lv.1 解锁每日任务 → Lv.100 解锁终极协议）

**关键设计**：
- 首次启动仪式 = 5-10 分钟完整黑屏体验，**不可跳过**
- 所有 UI 文案统一遵循词汇表
- 关键操作有"扫描"动画（不是 spinner）
- 月报告、季度复盘有专属仪式（背景音乐、动画、关键页面停留）

### 4.10 子系统 10：AI 底座系统

**职责**：分阶段构建 AI 能力。

**四个阶段**：
- **Phase 1（API 调用）**：Claude API 处理全部 AI 任务
- **Phase 2（RAG）**：本地向量库 + RAG 让 AI 检索全部历史
- **Phase 3（混合）**：本地小模型（Qwen 7B / DeepSeek 蒸馏）处理日常 + Claude 处理复杂
- **Phase 4（个人专属）**：基于聚合行为数据微调专属小模型

**关键设计**：
- 抽象 LLM 接口层，所有 Agent 通过统一接口调用，Phase 切换不影响业务代码
- Phase 2 的 RAG 是质变点：AI 第一次能引用半年前的对话
- Phase 4 微调使用 LoRA，硬件门槛低（消费级 GPU）

### 4.11 子系统 11：系统进化引擎（NEW）

**职责**：让系统自身随时间变得更懂宿主、更好用。

**核心组件**：
- Evolution Agent（每周一次后台运行）
- Agent 输出质量监控（用户对洞察的👍/👎、任务完成率、人格档案修正频率）
- **自动调整其他 Agent 的 Prompt 模板**（A/B 测试 + 渐进升级）
- 提议新规则、新工具、新触发条件
- 自我进化日志（透明可见，可回滚）

**关键设计**：
- Evolution Agent 是**唯一一个能修改其他 Agent 配置的 Agent**
- 每次修改记录到 system_evolution_log，用户可见且可回滚
- 修改前必须经过离线评估集验证（不能伤害已有能力）
- 用户可以查看"系统从上周到本周的进化"

### 5.2 Orchestrator 决策流程

```
用户消息 / 定时触发
        ↓
Orchestrator 注入：
- 宿主档案摘要（缓存）
- 最近 24h 事件流摘要
- 当前活跃目标
- 当前对话上下文
        ↓
LLM 决策（Sonnet）：
- 这是什么类型的输入？
- 需要哪些 Agent 协同？
- 调用顺序？
        ↓
执行 Agent 调用（并行或串行）
        ↓
Safety Agent 过滤
        ↓
返回用户 / 触发小人 / 写入数据
```

### 5.4 模型路由策略

```
路由规则（按优先级）：
1. 标注必需的模型 → 用标注模型
2. 输入 token > 50k → Opus 4.7
3. 涉及深度推理（复盘/洞察/进化）→ Opus 4.7
4. 涉及规划/对话主流程 → Sonnet 4.6
5. 简单分类/抽取/改写/小人台词 → Haiku 4.5
6. 默认 → Sonnet 4.6
```

**Prompt 缓存策略**（关键）：
- 宿主档案、长期目标、世界观 Prompt → 系统级缓存（命中率应 > 80%）
- 重复触发的日常 Prompt 模板（晨间规划、晚间复盘）→ 缓存模板部分
- 5 分钟缓存窗口适合交互；1 小时缓存窗口适合后台任务

## 6. 记忆与人格档案层

### 6.1 四类记忆

| 类型 | 存储 | 内容 | 检索方式 |
|---|---|---|---|
| **宿主档案** | SQLite JSON | 价值观/性格/激励偏好/红线/长期目标 | 直接查询，每次对话注入 |
| **事件流** | TimescaleDB / SQLite 时序表 | 行动/情绪/决策/对话/多源数据 | 时间窗+标签过滤 |
| **关系图谱** | SQLite + 图查询 | 目标-任务-人物-属性-属性源 | 图遍历 |
| **语义记忆** | sqlite-vec / pgvector | 事件/洞察/对话片段的向量 | 余弦相似度 |

### 6.2 记忆生命周期

```
原始输入（对话/录入/多源数据同步）
        ↓
   Safety Agent 过滤（去除危机/敏感内容标记）
        ↓
   实体抽取（Haiku 4.5）
        ↓
   写入事件流 + 向量化 + 关系图更新
        ↓
   按需聚合（每日 → 每周 → 每月）
        ↓
   遗忘策略（90 天后细节降级，保留摘要 + 关键洞察）
```

**遗忘策略详解**：
- 0-30 天：全细节保留
- 30-90 天：细节保留，但低价值事件（如"喝了水"）开始聚合
- 90-365 天：按周聚合，仅保留有标签的关键事件
- 365 天+：按月聚合 + 关键事件全保留 + 重大洞察永不丢

### 6.3 宿主档案的演化机制

- Profile Agent 每周一次扫描事件流，识别与档案矛盾的模式
- 矛盾出现时**提议更新**（不是直接改），用户在每周复盘时一键确认/拒绝
- 月度复盘 Opus 会做更深的人格扫描，可能提议添加新的红线、新的激励偏好
- 用户可随时打开档案查看 / 修改 / 删除任意条目（**数据主权**）
- 所有档案变更记录到 `profile_change_log`，可回滚

### 7.2 统一事件 Schema

所有数据源归一化到一个 Event 表：

```
Event {
  id: UUID
  user_id: string
  source: string              # e.g. "apple_health", "obsidian", "manual"
  type: string                # action / mood / decision / data / dialogue
  category: string            # health / work / learn / play / social / ...
  raw_payload: JSON           # 原始数据
  structured: JSON            # AI 结构化后
  embeddings: vector(1536)    # 语义向量
  occurred_at: timestamp      # 事件发生时间
  ingested_at: timestamp      # 数据进入系统时间
  confidence: float           # 数据可信度 0-1
  tags: string[]
  related_goal_ids: string[]
  related_task_ids: string[]
}
```

### 7.3 多源数据校验机制（关键）

实现"自欺识别"的工程基础：

```
场景：用户说"今天运动了 30 分钟"
        ↓
检索同时段数据：
- 苹果健康：心率数据显示静息 35 分钟
- Keep：无打卡记录
- 屏幕使用：手机被使用 28 分钟
- 地理位置：未离开家
        ↓
Decision Agent 综合判断：
- 多源数据均不支持"运动 30 分钟"
- 标记任务为"伪完成"（待确认）
- 触发小人对话："系统注意到一些不一致的信号..."
        ↓
用户解释或修正
        ↓
若用户坚持完成，可信度 -1
若用户承认，可信度 +0.5（诚实奖励）+ 任务标记真实状态
```

### 7.4 接入难度与处理策略

| 难度 | 策略 |
|---|---|
| 官方 API 友好（苹果健康/HealthKit）| 直接接入 |
| 官方 API 受限（微信/支付宝）| 本地数据库直读 / 用户手动导出 |
| 无 API（部分老应用）| 屏幕识别 + OCR / 抓包逆向 |
| 高敏感（财务/通讯）| 用户主动一键导入，端到端加密存储 |

## 10. 外星科技沉浸体系

### 10.1 世界观与文案统一

**世界观核心**：
> NEXUS-7 是一个来自未知文明的个体进化协议。
> 它选择宿主、绑定宿主、观察宿主、训练宿主。
> 系统的目标不是服务宿主的短期欲望，而是推动宿主完成长期跃迁。

**文案改写表**（见 0.4 词汇表）严格执行，所有 UI 文本、通知、小人对话遵守。

### 10.2 首次启动仪式（10 步）

```
1. 黑屏 + 心跳音效（3 秒）
2. 蓝色光点出现，扫描线扫过屏幕（5 秒）
3. 出现未知符号矩阵（3 秒）
4. 符号逐字转化为中文："检测到未绑定生命体"（4 秒）
5. 引导对话：宿主基础信息采集（性格、目标、激励偏好等）— 8 题
6. "正在分析认知模式..."（带进度条，实际在调用 Profile Agent）
7. 生成宿主档案 + 编号（NX-2026-xxxx）
8. 虚拟小人诞生动画（粒子凝聚 → Live2D 形象出现）
9. 系统给宿主的第一段话（Opus 生成，融合所采集信息）
10. 解锁第一层权限（每日任务）
```

整个流程 5-10 分钟，**不可跳过**。

### 10.3 UI 视觉语言

| 元素 | 设计 |
|---|---|
| 主色 | 深空黑 #0A0E1F + 量子蓝 #00D4FF + 警示红 #FF3D5A |
| 字体 | 等宽英文字体 + 中文衬线（强调）/ 中文黑体（正文）|
| 装饰 | 扫描线、网格、坐标系、数据流粒子 |
| 动画 | 所有进入/退出动画使用"扫描""解析""数据流"风格，避免普通 fade |
| 音效 | 关键操作有"扫描"音、"权限解锁"音；月报告有专属 BGM |
| 图表 | 全部自定义风格，不用默认 chart.js 主题 |

### 10.4 权限解锁阶梯

```
Lv.1 解锁每日任务
Lv.3 解锁能量商城
Lv.5 解锁每日校准（复盘）
Lv.10 解锁属性系统
Lv.15 解锁选择前预测
Lv.20 解锁每周总结
Lv.25 解锁多小人系统
Lv.30 解锁路径模拟（高级）
Lv.40 解锁多源数据接入
Lv.50 解锁深度人格分析
Lv.60 解锁黑箱裁决
Lv.80 解锁自主进化日志
Lv.100 解锁终极协议
```

**关键设计**：等级提升不是为了限制使用，而是创造**长期期待**。但 Phase 1 不能要求新用户立刻达到 Lv.5 才能复盘——所以前 5 级会迅速给（首启动仪式后立即 Lv.3）。

### 10.5 黑箱裁决机制

实现方式：
- 重大决策由 Opus 4.7 输出，包含"完整推理"和"压缩后的神秘表达"两版
- 系统根据"权限"决定显示哪版
- 用户达到 Lv.80 后可以看完整推理
- 关键安全决策（自伤、危机）必须可解释，不进入黑箱

## 13. 五阶段实施路线

### Phase 1 · 觉醒（0-3 个月）：MVP 跑通核心闭环

**目标**：宿主能完成"目标 → 任务 → 行动 → 复盘 → 反馈"完整闭环，主小人陪伴。

**包含**（共约 30 项原子功能）：
- 宿主档案（含仪式化首启动）
- 五层目标 + 模糊目标明确化 + 智能路径生成
- 任务系统（基础）
- 主小人 Live2D + Companion Agent + 桌面悬浮
- 11 个 Agent 全部 v1 实现（基础版）
- 记忆层（事件流 + 语义记忆 + Obsidian 同步）
- 手动录入 + 系统日历 + Obsidian 监听
- 6 维属性 + 觉醒等级
- 能量点 + 简单商城
- 每日复盘
- 外星科技 UI 主题 + 词汇表

**不包含**：多源数据校验、辅助小人、衰减机制、自欺识别、权限解锁、3D 升级、自主进化

- 系统宕机时间 < 1 小时/月

## 14. 开发方法论（AI 协同开发）

## 16. 数据安全与备份

### 16.1 个人项目的安全原则

不需要合规级隐私设计，但需要**对自己负责**：

1. **本地优先**：所有数据本地 SQLite/Postgres，**不上传任何云数据库**
2. **加密备份**：每日加密备份到 1 个对象存储（如 Backblaze B2）
3. **多机冗余**：开发机 + NAS + 云备份 三处
4. **敏感字段二次加密**：财务、私人通讯使用 age 二次加密
5. **API Key 管理**：1Password CLI + 环境变量
6. **数据导出**：随时可导出全部数据为 JSON + Markdown

### 16.2 备份策略

```
每日 03:00：
- SQLite/Postgres 全量 dump（age 加密）→ NAS
- Obsidian Vault 增量同步 → NAS
- 重要变更同步到云对象存储（B2）

每周日 04:00：
- 全量备份验证（restore 到临时数据库，跑 smoke test）
- 删除 90 天前的增量备份

每月 1 日：
- 归档备份（不可删除）
- 验证云端备份完整性
```

## 17. 核心指标体系

避免虚荣指标。所有指标围绕"AI 真的懂我，我真的成长"。

| 类别 | 指标 | 含义 | 目标 |

| | P95 响应延迟 | 体验门槛 | < 3 秒 |
