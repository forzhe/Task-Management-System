# 个人档案信息模块设计方案

以下仅针对**个人档案信息模块**做纯信息字段与展示功能设计，全程不涉及任务发布、奖惩执行、权限联动等其他系统模块的内容，所有信息均围绕 “量化每日用功状态、直观呈现成长轨迹” 的核心目标设置。

---

### 一、基础概览页（档案首页）

这是进入档案模块后默认展示的首页，仅呈现最核心的身份与当日用功概览信息。

- 基础身份字段：姓名、当前成长阶段（如高三备考、炼气期、Java 入门阶段）、核心用功目标（主角自主设定的长期目标，如 “考入目标院校”“突破筑基”“掌握全栈开发”）、系统累计绑定天数

- 核心状态标识：自律等级（仅作等级标识展示，由累计用功数据自然升级）、当日用功状态（实时更新，如「未开始 / 进行中 / 已达标 / 未达标」）

- 当日进度可视化：当日有效用功时长进度条（对应当日基础目标时长）、完成度百分比

### 二、用功数据明细档案

这是档案模块的核心数据区，分类存储全部用功相关的明细记录，支持按日 / 周 / 月筛选查看。

#### 1\. 每日用功明细

- 时长数据：当日总投入时长、有效专注时长（剔除低效走神时段后的净用功时长）、时段分布（上午 / 下午 / 晚间各时段用功时长占比）

- 内容分布：当日各科目 / 技能 / 功法的用功时长占比，对应具体的学习 / 修炼内容条目

- 质量数据：当日平均专注度（0\-100 分）、各分项内容的专注度对比、有效用功效率系数

- 当日完成清单：当日所有完成的用功条目明细，对应每条内容的投入时长与专注度

#### 2\. 分项能力熟练度档案

按用功方向独立分页存档，直观呈现每一项能力的积累进度：

- 基础信息：能力名称、当前掌握层级（了解→熟悉→掌握→精通→融会贯通）、当前熟练度数值、下一等级所需熟练度

- 累计数据：该能力累计投入有效时长、历史最高单日专注度、连续用功天数

- 成长记录：近 7 天 / 30 天的熟练度增长曲线，仅作数据展示

### 三、状态评估与诊断档案

基于存档的用功数据自动生成评估结论，全部为档案内的信息展示，不触发外部机制。

- 周期用功评级：按日 / 周 / 月生成综合评级（S/A/B/C/D 五档），附评级说明，标注本期核心数据达标 / 未达标的项

- 状态标签：根据近期数据自动生成状态描述，如「状态优异」「轻度懈怠」「效率下滑」「瓶颈滞缓」「偏科明显」

- 薄弱项标注：自动标记熟练度增长缓慢、投入时长不足、专注度偏低的项目，高亮展示在对应分项档案中

- 复盘结论：基于数据生成的中性评估总结，仅客观指出优势与不足，不附带强制任务或惩罚提示

### 四、成长轨迹存档区

永久存档的历史成长记录，用于回溯长期用功成果。

- 趋势曲线：支持切换日 / 周 / 月 / 年维度，展示有效用功时长趋势、专注度变化趋势、总熟练度增长趋势

- 里程碑档案：收录所有关键成长节点，如「首次单日有效用功 8 小时」「连续用功 30 天」「首个能力达到掌握层级」，附带对应日期与当时的核心数据

- 历史归档：支持查询任意日期的用功明细档案，历史数据不可修改

### 五、成就称号收录区

档案内的荣誉收录板块，仅展示已解锁内容，无额外功能属性。

- 自律坚持类：对应连续用功、累计用功时长的成就 / 称号

- 效率质量类：对应高专注度、高效率用功的成就 / 称号

- 能力进阶类：对应各能力等级突破的成就 / 称号

- 每个成就 / 称号附带解锁日期、解锁条件说明

> （注：部分内容可能由 AI 生成）
