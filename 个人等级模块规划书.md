# NEXUS-7 个人等级模块规划书

> 版本 v0.1 · 独立子系统规划（不并入整合书）
> 定位：把「等级」从一条孤零零的进度条，升级为宿主成长的**主仪表盘**——一眼回答「我现在处在觉醒的哪个阶段、强在哪、弱在哪、下一步要去哪」。
> 前置依赖（均已在现有代码就绪）：等级公式 `calcLevel` / `xpToNextLevel` ✅ · 六维属性 `AttributeSet` ✅ · 属性衰减 `DECAY_TIERS` ✅ · 解锁阶梯 `UNLOCK_LEVELS` ✅ · 可信度系统 §6.1 ✅ · 能量经济（`energyPoints`）✅

## 1. 核心命题

现状（见 [App.vue:1956-1991](apps/desktop/src/App.vue#L1956-L1991)）：等级页只有一个标题、一条 XP 进度条、六条属性横条。问题非常具体：

1. **像空页**。满级状态下六条属性全部顶到 9999、进度条全满，整页只剩三行有效信息，下方一大片留白。
2. **属性不可读**。每条属性只有「一个数字 + 一个被 `max(200, level*80)` 卡死的进度条」。看不出这个数字**意味着什么、从哪来、最近是涨是跌、有没有在衰减**。
3. **后端能力全被埋没**。系统其实已经有解锁阶梯、属性衰减、可信度、能量流水——但等级页一个都没暴露。宿主在这一页**学不到任何关于自己的东西**。

本模块回答一个问题：

> 当宿主打开「等级」页，他应该在 5 秒内知道——**我现在是谁、我在变强还是变弱、我接下来该往哪使劲**。

设计原则：**信息密度优先于装饰**。每一块面板都要么给出一个判断，要么给出一个方向。不放纯装饰元素。

设计铁律（宿主反馈）：**字号要够大、能看清，并与整页布局适配、美观**。现状页（截图）的属性名/数值挤在角落、字号偏小、在宽屏上大片留白——这是不可接受的。重设计必须做到：关键数字（等级/EP/可信度大数）显著放大，属性名与段位标签 ≥ 正文字号、清晰可读，卡片与面板**充分利用横向空间**、间距舒展，而非把内容压在一角。**信息密度不等于把字变小**——是用更大的字、铺满的布局，承载更多有效信息。

| 能量点 | 能量点（EP，可消费货币，接商城，会增减） |
| 数据可信度 | 可信度（0–2，数据质量与自律的信任分） |

| 成就/勋章 | 称号（Title） |
| 属性等级 | 属性段位 |
| 属性掉点 | 衰减（§8.1 已有） |
| 功能解锁等级 | 觉醒权限 / 解锁里程碑 |

> **关键澄清（现有代码已暗含但 UI 混淆）**：`totalExp`（觉醒经验，驱动等级）与 `energyPoints`（能量点，可消费）是**两个独立的量**。本模块第一件事就是把它们在视觉上彻底分开，不再都标成「EP」。

## 3. 三指标分工（页面顶部三张卡）

宿主答复明确：等级 / EP / 可信度**分工明确、并列展示**。三者各管一件事，绝不混为一谈：

| 指标 | 含义 | 性质 | 关联系统 | 卡片副文案 |
|---|---|---|---|---|
| **觉醒等级 Lv.** | 长期成长的总刻度 | 累计、单调不减 | 觉醒阶段 / 解锁 | 「你走了多远」 |
| **能量点 EP** | 可消费的行为货币 | 会增、会花 | 商城 / 心愿兑现 | 「你能兑换什么」→ 跳商城 |
| **可信度** | 对你数据真实度的信任分（0–2） | 随复盘自律波动 | 估值门槛 / 洞察信号 | 「系统多信任你的数据」 |

每张卡：大数字 + 一句话定义 + 一个**趋势角标**（↑ 本周 / → 持平 / ↓）。EP 卡可点击直达商城；可信度卡用现有 `credibilityColor` 配色（≥1.5 绿 / ≥1.0 中 / ≥0.5 橙 / 否则红）。

## 4. 属性分级 + 动态量程（核心机制升级）

宿主答复：采用**属性分级 + 动态量程**，根治「撞到 9999 硬顶」。

### 4.1 给每个属性一个独立段位

复用主等级的平方曲线思路，但系数更小，让属性升级节奏比主等级快一档：

```
属性段位 attrLevel(v) = floor( sqrt( v / 50 ) )      // 与 calcLevel 同构，K=50
段位 L 的门槛           = L² · 50
段位内进度 pct(v)       = (v − L²·50) / ((L+1)²·50 − L²·50)
```

**动态量程**：进度条永远显示「当前段位 → 下一段位」之间的填充，**与绝对值脱钩**，所以永不撞顶。

校验（值 = 9999）：`floor(sqrt(9999/50)) = floor(14.14) = 14` → 智力 **段位 14**；下一段位 15 门槛 = 225·50 = 11250；进度 = (9999−9800)/(11250−9800) ≈ **14%**。满级宿主的属性条不再是平的死条，而是「段位 14，距 15 还差 1251 点」。

### 4.2 属性掌握度命名（可选叠加层，建议保留）

给属性段位再叠一层语义标签，让数字有「身份」：

| 段位区间 | 掌握度标签 |
|---|---|
| 0–4 | 萌芽 |
| 5–9 | 熟练 |
| 10–14 | 精通 |
| 15–19 | 卓越 |
| 20+ | 大师 |

### 4.3 属性详情与来源追溯（点击展开）

每个属性行可展开，给出**四件可读信息**：

- **含义**：一句话——「智力＝深度学习与认知产出的积累」。
- **最近变化**：本周 ↑+12 / 7 日趋势迷你折线。
- **来源追溯**：贡献最大的近期行为（「来源：深度任务 ×3、复盘 ×1」）。需要在 EP/属性结算时打来源标签（见 §8）。
- **衰减状态**：直接复用 `AttributeMeta.tier`（0 正常 / 1 = 7 天 / 2 = 14 天 / 3 = 30 天）+ `DECAY_TIERS` 比例。tier ≥ 2 时显示「⚠ 荒废中，已衰减 −10%」并给关怀提示。

| 属性 | 含义（页面文案草案） | 主要来源行为 |
|---|---|---|
| 智力 intellect | 深度学习与认知产出 | 深度任务、复盘洞察 |
| 体力 stamina | 身体维护与精力基线 | 运动、作息、健康源 |
| 专注力 focus | 单位时间专注质量 | 番茄钟、无打断深度块 |
| 意志力 willpower | 抗诱惑与坚持难事 | 完成困难/拖延任务、守红线 |
| 创造力 creativity | 创造性产出与发散 | 创作类任务、新尝试 |
| 秩序感 order | 计划性与生活整洁度 | 按计划完成、习惯链不断 |

## 5. 觉醒阶段 + 称号系统

### 5.1 觉醒阶段（Awakening Stage）

把等级带分段命名，对齐既有的解锁里程碑（`UNLOCK_LEVELS`：3/5/10/15/20/30/50/100），让「升级」有叙事感而非只是数字 +1。以下为**草案，命名可调**：

| 阶段 | 等级区间 | 阶段名（释义） | 叙事 / 标志解锁 |
|---|---|---|---|
| Ⅰ | Lv.1–4 | 启灵（灵识初启） | 核心循环跑通，能量点商城将启（Lv.3） |
| Ⅱ | Lv.5–9 | 见独（照见自身） | 日终校准与复盘洞察上线（Lv.5） |
| Ⅲ | Lv.10–14 | 赋形（属性显形） | 六维属性面板解锁（Lv.10） |
| Ⅳ | Lv.15–19 | 守中（持中校准） | 行为影响评分（Lv.15） |
| Ⅴ | Lv.20–29 | 同波（节律共振） | 每周总结（Lv.20） |
| Ⅵ | Lv.30–49 | 衍象（推演万象） | 人生路线模拟（Lv.30） |
| Ⅶ | Lv.50–99 | 自在（自治自如） | 深度长期趋势（Lv.50） |
| Ⅷ | Lv.100+ | 盟约（终极协议） | 终极协议 / 终端（Lv.100） |

页面顶部大等级环旁显示**当前阶段名 + 距下一阶段还差多少级**，给出「再升 N 级进入『同波』」的牵引。

### 5.2 称号（Title）

比阶段更细、由**具体行为**赚取的徽章，是个性化炫耀位（顶部可佩戴一个当前称号）。称号判定全部基于**现有数据**，不需要新行为埋点：

| 称号 | 解锁条件 | 数据来源 |
|---|---|---|
| 心流匠人 | 专注力段位 ≥ 精通（≥10） | 属性段位 |
| 铁律者 | 任一习惯链 longestStreak ≥ 21 | `UserStreak` |
| 均衡之主 | 六维属性极差（max−min 段位）≤ 2 | 属性段位 |
| 长跑选手 | 累计等级达 Lv.30 | currentLevel |
| 诚实之证 | 可信度连续 N 周 ≥ 1.5 | credibilityScore |
| 不破之链 | 30 天内无习惯链断裂 | `UserStreak.brokenAt` |

> 称号只读、不可购买（与 §6 估值定价铁律一致：身份由行为赚取，不由能量点买）。

## 6. 解锁路线图

把 `UNLOCK_LEVELS` 从一个隐藏的 map 变成一条**可视化路线**：

```
Lv.3 商城 ✓ ── Lv.5 复盘 ✓ ── Lv.10 属性 ✓ ── Lv.15 评分 ✓
   ── Lv.20 周报 ✓ ── Lv.30 路线模拟 ✓ ── Lv.50 深度趋势 ✓ ── Lv.100 终极协议 ○
                                                              还差 N 级 / 约 M XP
```

- 已解锁节点：实心 + 勾。
- 下一个未解锁节点：高亮 + 「还差 N 级（≈ M 觉醒经验）」，M 由 `xpToNextLevel` 链式累加算出。
- 给低等级宿主明确的「下一个奖励在哪」，给高等级宿主「就剩终极协议」的收尾目标感。

## 7. 等级历程 / EP 时间线

页面底部一条**可回溯的成长史**，由事件流（`NexusEvent`）聚合：

- **里程碑流**：每次升级、每次进入新觉醒阶段、每个新称号，按时间倒序成卡。
  - `06-20  升至 Lv.50 · 进入「自在」阶段`
  - `06-18  解锁称号「铁律者」`
- **EP 来源构成**：近 7 / 30 日能量点来自哪里（任务 / 目标达成 / 复盘 / 兑现支出），用一条堆叠条 + 列表呈现，让宿主看清「我的能量从哪赚、往哪花」。

> 实现前提：升级 / 阶段跃迁 / 称号需要**落一条事件**；EP 进出需要**带来源标签**（见 §8 数据改动）。

## 11. 铁律与开放问题

**铁律**
1. 称号与阶段是**赚来的身份**，不可用能量点购买（与商城估值铁律一致）。
2. 觉醒经验（等级）单调不减；衰减只作用于**属性**，不回收等级。
3. 所有新公式（属性段位、阶段、称号）落 `packages/shared` 单一真源，前端不另算一套。
4. **字号与布局适配美观**（§1 设计铁律）：关键数字显著放大、属性可读、铺满横向空间，绝不重蹈现状「字小挤角、宽屏留白」。

**已拍板的决策**
- ✅ **称号**：顶部只佩戴一个主称号；另设可展开的「称号墙」查看全部已获得（§5.2）。
- ✅ **衰减关怀**：平时在属性展开里被动显示；仅当某属性**新跨入 14 天荒废**（复用 `AttributeDecayResult.needsCare`）时，在属性区顶部主动弹一次提示，不重复打扰。
- ◽ **属性段位系数 K=50**（§4.1）：暂定 50，作为可调参数保留（K 越小升级越快、数字越好看；可后续调 30/50/80），不阻塞开工。
- ✅ **觉醒阶段命名**（§5.1）：采用宿主定名——启灵 / 见独 / 赋形 / 守中 / 同波 / 衍象 / 自在 / 盟约，已对齐等级区间与解锁里程碑。
