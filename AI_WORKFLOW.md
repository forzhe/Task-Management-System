# AI_WORKFLOW · NEXUS-7 人机协作记录

> 本文综合记录这一轮 NEXUS-7（AI 驱动的个人人生优化系统）开发协作的全过程，
> 按「原始需求 → 我给 AI 的任务拆解 → AI 的方案 → 我发现的问题 → 我补充的测试 → 最终取舍」六个维度归档，供回顾与复用。
> 定位始终不变：**NEXUS-7 是 AI 人生优化系统**，核心命题——「今天的行为让宿主更接近还是更远离理想人生？」。

## 1. 原始需求

按时间顺序，我（宿主）先后提出：

1. **前端优化**：「优化前端，包括页面显示、布局……都挤在一起了，太丑，不切合项目的酷觉，其他不要动。」
2. **抽屉化**：「做成抽屉样式，不要全部平铺放在一个页面上。」
3. **离线疑问**：「为什么还都是离线状态？系统不能使用吗？」并追问觉醒仪式怎么触发、跳过后怎么重来。
4. **兜底问题**：贴了晨间规划/日终复盘两张截图——文字密密麻麻、且出现「结构化输出不可用，使用最小闭环兜底计划」，要求「修复这个问题」。
5. **13 条体验问题**：整理进 `使用过程的问题.md`（觉醒仪式体验、AI 语气、新手引导、等级可见、排版、晨间频率、数据管理、证据图片、性格分流、满级测试号……），要求逐条解决。
6. **全局反馈**：「点击所有的按钮，不要让使用人空等待，最好有一个进度条或者提醒方式。」
7. **学习对标**：「检查一个 app 叫番茄ToDo，学习它的产品逻辑、设计美学等」——随后澄清：**定位不变**，只学「页面/视觉优化、降密度」。
8. **视觉克制**：「先做一件事——更克制的科幻（深色但留白、降密度）。」
9. **聊天小窗**：「神经链接·实时通信做成小窗，不要固定。」
10. **首页图形化**：「首页改为各种图形汇总，一进来就用直观的数据图、分析图告诉我是否在前进。」
11. **视图拆分**：「原来的工作区另起一个侧边按钮实现切换，不要再出现首页了。」

## 2. 我给 AI 的任务拆解（约束与决策）

我对每件事的边界与取向都给得很明确，AI 据此执行：

- **范围收口**：「其他不要动」「只做这一件事」——多次强制单点、避免顺手扩张。
- **关键分叉用选项确认**（AI 用 AskUserQuestion，我来拍板）：
  - 抽屉形式 → 选「左栏 + 侧滑抽屉」（而非顶部标签 / 折叠面板）。
  - 离线修复 → 选「修好 .env 加载，联网用 DeepSeek」+「清空数据库、全新开始」。
  - 13 条节奏 → 选「全部做·分三批交付」；满级号选「pnpm 脚本」；#3 确认为「觉醒问卷太长」。
  - 番茄ToDo 之后 → 选「先不动手，再谈」，再单独指定「先做视觉克制」。
- **分批验收**：明确「每批做完我先测一轮，确认 OK 再开下一批」。
- **理想态留白**：我直说「理想的样子很难描述，因为想实现的功能在不断调整」——所以采用**小步快迭代**而非一次性大重构。

## 3. AI 的方案

按主题归纳 AI 实际给出并落地的方案（文件链接为相对路径）：

- **布局演进**：单列堆叠 → 多列瀑布流 → **左栏 + 右侧侧滑抽屉**（9 模块入口，一次只开一个）→ 再叠加**主视图切换**（总览/执行互斥）。核心在 `apps/desktop/src/App.vue` + `apps/desktop/src/styles.css`。
- **离线根因修复**：`apps/server/src/main.ts` 启动时加载仓库根 `.env`（dev:server 工作目录在 `apps/server`，原本读不到根 `.env`），让 `NEXUS_LLM_PROVIDER` / API key 生效。
- **兜底根因修复**：DeepSeek 模型名 `deepseek-v4-pro`（不存在）→ 改 `deepseek-chat`，同时修正 `packages/ai-core/src/llm.ts` 的默认值；清理抢占 3737 端口的重复 server。
- **13 条 · Phase A（快赢，纯前端+提示词+脚本）**：觉醒仪式星空背景/合成氛围音/预设胶囊/题目 hint/字数限制（`OnboardingRitual.vue`）；新手指引 `GuideTour.vue`；顶栏等级胶囊；仪式结果改结构化卡片；晨间一天一次守卫 + 系统日期；提示词软化 + 接 `personalityPreference`（`prompt-registry.ts` / `orchestrator.ts`）；满级脚本 `scripts/seed-maxed.ts`（`pnpm seed:maxed`）。
- **13 条 · Phase B（中等，新增后端+UI）**：数据管理抽屉（按日期看/删任务·事件·复盘）；任务行内编辑/删除；晨间重做 + 模型档位切换（贯通 `chat/send` → `orchestrator` → `planning-agent` → `LlmRequest.model`）。后端动 `repository.ts` / `nexus.service.ts` / `app.controller.ts`。
- **13 条 · Phase C（产品）**：性格测评抽屉 → 写 `profile.traits` 并注入 planning/companion 提示词使方案因人而异；证据图片上传（Nest 静态托管 `/uploads` + `POST /uploads` base64 落地，前端缩略图预览）。
- **全局反馈**：在唯一请求出入口 `apps/desktop/src/api.ts` 的 `request()` 加在途计数，驱动顶部进度条 + 「处理中…」toast（180ms 隐藏防抖防闪烁）。
- **视觉克制**：令牌层（边框降到发丝级、嵌套面由暗块改极淡提亮、阴影减轻、底色色晕收敛）+ 间距留白（外壳/面板/抽屉/任务卡）。
- **聊天小窗**：抽出 `apps/desktop/src/components/ChatWindow.vue`，浮动可拖动 + 最小化/关闭 + 位置记忆 + 唤起按钮；栅格 `events` 补位铺底。
- **首页图形总览**：`App.vue` 顶部 `.overview` 仪表盘——净成长 hero（是否在前进）/ 跨周趋势 sparkline / 六维雷达 / 等级 / 习惯链 / 周期，复用既有 `sparkline()`、`levelInfo`、`attrPct`、`streakChips`。
- **主视图切换**：左栏顶部「总览 / 执行」两个按钮（青色高亮，区别于模块抽屉的琥珀色），`overview` 与 `workspace` 互斥，选择存 `localStorage`。

## 4. 我发现的问题

我以「真实使用者」视角，反复指出 AI 没覆盖到或方向不对的地方：

- **离线不可用**：配好 DeepSeek 仍显示「离线模拟」——暴露出 `dev:server` 根本没读 `.env`（AI 一开始只跑前端，截图里的「Failed to fetch」也误导）。
- **兜底降级**：晨间规划弹「结构化输出不可用」——暴露出模型名写死成不存在的 `deepseek-v4-pro`，每次 400 静默回落兜底。
- **端口冲突**：`EADDRINUSE :::3737`——同时跑了两个 server。
- **体验细节（13 条）**：觉醒仪式手写太多/纯黑背景/没音乐；AI 语气「不讨好只诚实」过冷；进去不知干嘛、没新手引导；看不到自己几级；晨间/复盘文字堆叠丑；晨间能一天点多次；不能改/换模型重生成；不能管理某天数据；证据不能传图……
- **产品方向**：让我去研究番茄ToDo，并**纠正 AI 的理解**——不是要转型成番茄钟工具，定位仍是人生优化系统，只借鉴「降密度、留白、克制」。
- **交互固化**：聊天面板固定在栅格里不灵活 → 要浮动小窗。
- **首页信息组织**：一进来不该堆面板 → 要图形化「是否在前进」；工作区与总览不该挤在一页 → 拆成独立视图。

## 5. 我补充的测试

测试由「我的真实使用反馈」和「AI 的自动化验证」两部分共同构成：

**我贡献的测试 / 验收：**
- 整理 `使用过程的问题.md`（13 条来自亲自体验，含两张界面 OCR 截图作为证据）。
- 要求并使用**满级测试账号**（`pnpm seed:maxed` → Lv.60、六维拉满、全模块解锁），以便覆盖到高等级才解锁的功能。
- **分批验收**：每个 Phase 交付后我先手测一轮、确认手感，再放行下一批。
- 持续以使用者视角抛出问题（离线、兜底、密度、固定面板、首页组织）。

**AI 配套的验证手段：**
- 每次改动后 `vue-tsc` 类型检查 + `vite build`；后端 `tsc` 类型检查。
- 全套单测：`ai-core 25 · memory 12 · server 16 · desktop 2` 全过。
- Playwright 截图核对关键界面（抽屉、仪式、卡片排版、数据管理、上传缩略图、进度条/toast、首页雷达/趋势、视图切换）。
- 联调验证：`/health` 确认 `offlineLlm=false / llmProvider=deepseek`；真实跑一次晨间规划确认 `fallbackUsed` 为否、不再回落兜底。

## 6. 最终取舍

- **定位不变**：拒绝把产品转型成番茄钟，保持「AI 人生优化系统」；番茄ToDo 的经验**只用于视觉/信息组织**（降密度、留白、克制），不动核心命题与功能架构。
- **布局路线**：masonry → 抽屉（左栏+侧滑）→ 主视图切换（总览/执行互斥）。以「一次只暴露一组信息」替代「全部平铺」。
- **视觉**：保留深色科幻，但更克制（发丝边框、嵌套面淡提亮、阴影减轻、留白）。浅色模式本轮不动（聚焦深色）。
- **AI 语气**：软化「不讨好/不软化」一类口号，但**保留「如实指出偏差」的诚实内核**；首次把觉醒仪式采集的 `personalityPreference`（strict/gentle，默认 gentle）真正接进提示词。
- **模型切换**：provider 固定取自 `.env`（当前仅 DeepSeek）；「换模型」只能在已配置的档位/模型间切（含 `deepseek-reasoner`），不凭空新增需密钥的 provider；重做本身（同模型）也会产出不同方案。
- **环境加载**：选择在 `main.ts` 启动时加载根 `.env`（代码内置、稳健），而非改 `tsx` 启动脚本。
- **数据策略**：离线修复时按我的选择**清库全新开始**；满级号用**可重复运行的脚本**而非 UI 按钮（不污染界面、仅本地）。
- **首页与工作区**：默认进来是**图形总览**；工作区移到独立「执行」视图，二者互斥、选择持久化——首页不再与工作区同屏。
- **浮窗常驻**：聊天与系统核心都做成可拖动浮窗、位置记忆，跨「总览/执行」两视图都在。
- **工程铁律**：全程 TypeScript strict、UTF-8；Agent 数据访问只走 `NexusTools`，LLM 只走 `LlmClient` / `ModelRouter`；不提交真实 token、不引入大二进制（氛围音用 Web Audio 合成）。

## 协作方式小结

- **单点收口 + 小步快迭代**：每轮只动一件事，先确认边界再下手，避免在「需求还在调整」时做大重构。
- **关键分叉显式确认**：涉及花钱（联网/换模型）、丢数据（清库）、大方向（产品转型）时，先给选项让我拍板，再执行。
- **真实使用驱动**：以满级测试号 + 亲自体验为主要发现问题的方式，AI 用类型检查/单测/截图做客观回归。
- **诚实优先**：定位、模型限制、范围注脚都如实标注，不掩盖取舍。
