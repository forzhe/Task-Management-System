# NEXUS-7 宿主档案子系统规划书

> 版本 v0.1 · 独立子系统规划（不并入整合书）
> 定位：宿主档案从「静态资料表单」升级为「NEXUS 对『你是谁』的活体自我模型」——**声明 / 观测 / 裁决** 三层闭环。
> 前置依赖（均已在现有代码就绪）：档案演化 §5.3 ✅ · `ProfileAgent` + `profile_change_log` ✅ · 本地事件流 + 日终校准 ✅ · 多 Agent 编排 §6.7.3 ✅ · 性格测评 `traits` / 六维属性 ✅
> 本次共定的设计决策：① MBTI＝**自评定基线＋行为校准** ② 画像节奏＝**日终汇总＋随时手动深扫** ③ 演化权责＝**低风险自动／高风险请示** ④ 信息架构＝**分层展示（核心常驻＋画像可展开）**

---

## 1. 核心命题

现状：档案页是一张静态表单（代号 / 专注 / 愿景 / 红线），且只在「编辑」时可见，平时近乎空白；系统对宿主的"了解"散落在事件流里，从未沉淀成一份**可读、可演化的自我画像**。

本子系统回答一个问题：

> NEXUS 眼里的"你"，此刻是什么样子？这个画像，比昨天更准了吗？

档案不再是你填给系统的表格，而是 **系统与你共同维护的一份"自我模型"**：
你声明你想成为谁（声明层），系统观测你实际是谁（观测层），两者的差距，就是成长的坐标（裁决层）。

哲学锚点（与商城子系统对仗）：

> 商城：数据主权属于宿主，**定价**主权属于系统。
> 档案：观测主权属于系统，**身份**主权属于宿主。

系统可以无限地*观测*你、*侧写*你、*提议*你演化——但「你是谁」，最终由你裁决。

## 2. 铁律（禁区⑧）

> **禁区⑧：身份主权属于宿主。系统可观测、可提议，但不可改写宿主的自我定义。**

1. 档案分**声明层 / 观测层 / 裁决层**（§4）。声明层由宿主独占编辑；观测层由系统独占写入、对宿主**只读**；任何"观测 → 声明"的回写，只能走裁决层。
2. **观测层不是结论，是证据**。系统对你的人格 / 动机 / MBTI 的判断，永远以「置信度 ＋ 证据样本数」形式呈现，并标注「这是 NEXUS 的看法，不是定论」。
3. **演化分级**（本次决策 ③）：
   - **低风险观测字段**（作息节律、执行风格、专注模式、动机倾向的*观测值*）→ 系统可在日终**自动刷新观测层快照**，事后告知，宿主可一键回滚。
   - **高风险声明字段**（红线、长期愿景、姓名 / 代号、MBTI 定型）→ 永远只生成**演化提议**，宿主裁决后才生效。
4. 红线享**最高保护**：系统不得自动新增 / 删除 / 改写任何一条红线，只能提议；红线物同时受商城禁区⑦约束（拒绝纳入悬赏经济）。
5. **全程可回溯、且可遗忘**：每次观测快照、每条演化提议都落 `profile_change_log` / 事件流，宿主可查「为什么 NEXUS 这么看我」；宿主有权**一键清空观测层并重算**（数据主权兜底）。

## 4. 三层模型架构

```
                  ┌───────────────── 宿主档案（自我模型）─────────────────┐
   宿主编辑 ─────▶ │  声明层 Declared        裁决层 Reconcile        观测层 Observed │ ◀───── 系统写入
                  │  姓名/代号              演化提议 → 宿主裁决        活体画像快照     │
                  │  当前专注                     ▲                   行为人格证据     │
                  │  长期愿景                     │ 矛盾/漂移触发        作息·动机·风格   │
                  │  红线协议 🔒                   └──────────────────  MBTI 行为证据    │
                  │  MBTI 基线(自评)                                   红线契合度       │
                  └──────────────────────────────────────────────────────────────────┘
```

### 4.1 声明层 Declared（宿主所有，可编辑）

| 字段 | 说明 | 落点 |
|---|---|---|
| 姓名 / 称呼 | 真实姓名（可空），用于人称化语气 | `basicInfo.name` |
| 代号 Call Sign | 系统内昵称 | `basicInfo.codename` |
| 当前专注 Directive | 此刻最重要的一件事 | `basicInfo.focus` |
| 长期愿景 Vision Vector | 日终「净增长」的标尺 | `longTermVision.statement` |
| 红线协议 Hard Constraints 🔒 | 不可触碰的底线（每行一条） | `redLines[]` |
| MBTI 人格基线 | 自评类型，作为校准起点 | `traits.mbti = { type, source:'self' }` |
| 动机自述 | 我认为自己被什么驱动 | `motivations.declared` |

### 4.2 观测层 Observed（系统所有，对宿主只读）＝ 实时用户画像

系统从事件流提炼出的「活体画像」，每日刷新一次快照（§6）。维度见 §6.2。
每个判断都带 `confidence`（0–1）与 `sampleN`（证据条数），低于阈值时显示「证据不足，继续观察中」。

### 4.3 裁决层 Reconcile（共治）

当观测层与声明层产生**矛盾或显著漂移**（例：愿景写「早起」，观测连续 14 天凌晨活跃；自评 INTJ，行为证据强烈指向 ENTP）时，侧写师生成**演化提议**，进入 `profile_change_log`，由宿主接受 / 否决（§8）。

## 5. 数据模型（扩展现有 `Profile`，不破坏现状）

现有 `Profile` 字段保持不变，仅**语义归层 ＋ 新增观测快照**。观测层**复用事件流**存储，不新建表：每日一条 `category = "profile_observation"` 事件，最新一条缓存到 `profile.observed`，趋势由事件序列计算。

```ts
interface Profile {
  userId: string;
  basicInfo: {              // ── 声明层
    name?: string;          // 新增
    codename: string;
    focus: string;
  };
  longTermVision: { statement: string };   // 声明层
  redLines: string[];                       // 声明层 🔒
  motivations: { declared?: string };       // 声明层
  traits: {
    mbti?: { type: string; source: "self"; setAt: string };  // 人格基线（自评）
    rhythm?: string; motivation?: string; workStyle?: string; approach?: string; // 既有短测评
    personaSummary?: string;
  };
  observed?: ProfileObservation;            // ── 观测层（最新快照缓存，新增）
  updatedAt: string;
  version: number;
}

// 观测层快照（每日落事件流；profile.observed 缓存最新一条）
interface ProfileObservation {
  takenAt: string;
  dimensions: Record<ObservedDim, { score: number; confidence: number; sampleN: number; note?: string }>;
  mbtiEvidence: { EI: Axis; SN: Axis; TF: Axis; JP: Axis }; // 行为证据，独立于自评基线
  redLineFit: { line: string; adherence: number; breaches: number }[]; // 红线契合度
  netGrowthVerdict: "closer" | "neutral" | "further"; // 对照长期愿景：更近/持平/更远
  source: "daily" | "deep-scan";
  engine: "llm" | "deterministic";
}
type ObservedDim = "rhythm" | "focus" | "drive" | "execution" | "discipline" | "creativity";
interface Axis { lean: string; score: number; confidence: number; sampleN: number } // 如 lean:"E", score:0.62
```

## 6. 活体画像引擎（观测层 · 日终汇总 ＋ 手动深扫）

### 6.1 运行节奏（本次决策 ②）

| 触发 | 模型 | 行为 |
|---|---|---|
| **日终校准时**（自动） | sonnet / 确定性 | 汇总当日事件 → 增量更新画像快照，落 `profile_observation` 事件 |
| **宿主点「立即分析」**（手动深扫） | opus | 取近 28 天长窗事件做一次深度侧写，刷新快照并可触发演化提议 |
| 离线 / 无 API | 确定性公式 | 各维度由可量化信号直接算分，保证可跑、可测（`engine:"deterministic"`） |

> "实时" ＝ 每日自动刷新 ＋ 随时可手动深扫。不做事件级逐条刷新（成本与抖动不划算），但画像永远不超过 24 小时陈旧。

### 6.2 画像维度（六维雷达，与六维属性区分：属性是「能力」，画像是「模式」）

| 维度 | 含义 | 主要信号 | 确定性兜底算法 |
|---|---|---|---|
| 作息节律 rhythm | 精力高峰落在何时 | 事件 `occurredAt` 时段分布 | 完成/专注事件的小时直方图峰值 |
| 专注模式 focus | 深度专注 vs 碎片切换 | 番茄钟时长、任务切换频率 | 平均专注段长 / 中断次数 |
| 驱动来源 drive | 成就 / 好奇 / 认可 / 安全 | 任务类型、悬赏动机、复盘措辞 | 各动机标签事件占比 |
| 执行风格 execution | 冲刺爆发 vs 稳定持续 | 任务完成的时间集中度 | 完成时间的基尼系数 |
| 自律强度 discipline | 计划兑现率、红线契合 | 计划 vs 实际、红线触碰 | 已完成/已承诺比率 |
| 创造倾向 creativity | 探索/产出新内容的比重 | 创作类任务、灵感事件 | 创作类事件占比 |

### 6.3 引擎分层

- **确定性层**先算出每维分数与样本数（永远可跑，是护栏）。
- **LLM 侧写层**（侧写师 Agent）在确定性结果之上做**解读与措辞**，输出受校验 JSON；越界（分数超出确定性 ±0.2）即丢弃，回退确定性值。
- 输出**必须解析为受校验 JSON 才落盘**（工程铁律）。

## 7. MBTI 子系统（自评定基线 ＋ 行为校准）（本次决策 ①）

### 7.1 四轴

`E/I（外向-内向）· S/N（实感-直觉）· T/F（思考-情感）· J/P（判断-感知）`

### 7.2 基线：自评（声明层）

宿主在档案里选定或做 8–12 题短测评 → 写入 `traits.mbti = { type, source:"self" }`。这是**起点**，不是定论。

### 7.3 行为证据累积（观测层，与基线并行、互不覆盖）

每日画像快照里维护 `mbtiEvidence`：每轴一个**带样本数的滑动估计**。证据信号映射示例：

| 轴 | 偏向证据 |
|---|---|
| E/I | 主动发起对话 / 社交类任务 ↑→E；独处深度工作时长 ↑→I |
| S/N | 具体清单步骤 ↑→S；目标抽象化、灵感跳跃 ↑→N |
| T/F | 复盘措辞重逻辑/指标→T；重感受/关系→F |
| J/P | 计划兑现、提前完成→J；临场调整、deadline 冲刺→P |

### 7.4 校准 → 演化提议（高风险，必请示）

当某轴**行为证据**与**自评基线**方向相反、且 `confidence ≥ 0.7 且 sampleN ≥ N₀`，侧写师生成一条**演化提议**：

> 「自评 I，但近 21 天有 18 次主动社交发起（置信 74%）。是否把人格基线更新为 E？」

MBTI 定型属高风险字段 → **绝不自动改**，永远走宿主裁决（§8）。

## 8. 档案演化裁决（低风险自动 ／ 高风险请示）（本次决策 ③）

### 8.1 风险分级表

| 风险级 | 字段 | 系统权限 |
|---|---|---|
| 🟢 低风险 | 观测层全部维度分数、作息/专注/风格/动机的*观测值* | **自动刷新**，告知，可回滚 |
| 🟡 中风险 | `personaSummary`、动机标签主次序 | 自动刷新 ＋ 高亮提示「已更新，点此查看」 |
| 🔴 高风险 | 红线、长期愿景、姓名/代号、MBTI 定型 | **只提议，必裁决** |

### 8.2 提议状态机

```
OBSERVED(矛盾/漂移) ──侧写师生成──▶ PROPOSED ──┬─(宿主接受)─▶ APPLIED（写入声明层，version+1）
                                              ├─(宿主否决)─▶ DISMISSED（记录，短期内不再就同一漂移提议）
                                              └─(超 30 天未裁决)─▶ EXPIRED
```

### 8.3 自动更新的护栏

- 自动刷新仅限 🟢/🟡，且每字段**每日至多一次**、变动幅度封顶（防抖动）。
- 每次自动更新写 `profile_change_log`（`auto:true`），档案页「演化时间线」可见、可一键回滚到任一历史快照。

- **hero 偏差带（常驻头条）**：身份卡下方一条带 —— **周期评级 S/A/B/C/D ＋ 状态标签 ＋ 最弱项 ＋ 净增长方向**，把「方向对不对、哪里在漂、要不要校准」做成第一眼信号。评级**对齐导向**：净增长趋势 50%（主），自律 20% ＋ 连续力 15% ＋ 可信度 15%（辅），**刻意不计用功时长，规避机械打卡**。纯函数 `assessProfileGrade`（`@nexus/shared`，3 项单测）。
- **核心常驻**：身份卡 ＋ 声明层（愿景/专注/动机/红线）一屏可见——直接解决「平时页面空白」。
- **画像可展开**：观测层默认折叠为一行摘要（「今天比昨天更近 ↑ · 自律 72%」），展开见雷达 ＋ 趋势 ＋ MBTI 证据。
- **裁决区独立**：高风险提议与可回滚时间线分区呈现，权责一眼分明。
- 复用本次已落地的暗色 HUD dossier 视觉语言（角标、等宽英文副标签、印记、脉冲）。

## 10. 角色：侧写师 Agent（扩展现有 `ProfileAgent`）

| 辅助 Agent | 职责 | 数据依赖 | 解锁 |
|---|---|---|---|
| 侧写师 Profiler | 日终画像汇总、手动深扫、MBTI 行为校准、矛盾检测与演化提议 | 全源事件流 ＋ 声明层 ＋ 历史快照 | Lv.5（与洞察同级） |

- 独立 `personalityParams`（观察者气质：克制、精准、不下定论、措辞像侧写而非评判）。
- 接 Orchestrator 场景路由：`日终校准` / `手动深扫` 触发；输出经主小人汇总，不另开聊天框。
- **边界**：输出落盘前**必须解析为受校验 JSON**；离线退回确定性画像；对人格判断一律附置信度，**禁止做医疗/心理诊断式断言**。

## 13. 安全 / 隐私护栏

1. 观测层数据**本地优先**，不出本机；画像不写入 Obsidian 公开副本（除非宿主显式导出）。
2. 宿主可**一键清空观测层并重算**，或导出/删除全部档案数据（数据主权）。
3. MBTI / 人格判断**仅作行为模式描述**，措辞回避诊断、标签化、宿命论。
4. 红线内容只在本地参与日终对照，绝不外发。
